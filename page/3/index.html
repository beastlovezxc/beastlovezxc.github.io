该死的自制力

一直以为自己是个自制力很强的家伙，起码从小学开始，想要做的事情坚持到底，想到的决策贯彻下去，成绩很好，爱好广泛，还蛮受欢迎的

看看现在的自己，每天浑浑噩噩，想学的技术，想健身塑型，想养成良好的作息想一切对自身有利的习惯，都坚持了半个多月然后就中断掉。被漫画，被游戏，被电影，被电视剧打断。每天憋在寝室，头发不剪，衣服不洗，蓬头垢面，活脱脱一个Loser的真实写照。

心血来潮给租了服务器，搭个博客，立志每天一篇技术博文，重新构建自己的github,好好学习ios开发，然后博客搭成了，博文却没有。github弄好了，fork了那么多好的开发库，却不怎么去看，ios的书买了四五本，看完两本就看不下去了。

说到底，还是没有自制力。

书看着看着就想翻微博，翻知乎，即便知道没有什么新内容也无可救药的翻。每当要努力看书就要脱到某个时间点借口给自己调整的时间。每天真正汲取新知识的时间只有不到两个小时，每晚打水的时候看着别人准备考研的学的筋疲力尽的回到宿舍，只有深深的浪费时间的自责感。

说到底，我还是不满足于自己的现状的。

我还是希望自己能够改变回去，有目标，有执行力，能控制自己。

每天早上7点起床，吃早餐，看书，写代码，然后午餐，午睡，健身房，再然后晚餐，看书，写代码，写博客。我希望我每天都过的充实一点，每天都能提高一点。

我不知道自己是不是又要坚持个几天就放弃掉，那就来试试吧，先坚持到月底。

发表于 2015-12-16   |   分类于 杂谈   |  

社交网络

堕落了好久，有一周时间了吧，没怎么看书，每天都在刷知乎，微博，看斗鱼上的赌片直播，大四保研烂在寝室的感觉就是这样了，无意间看到了《社交网络》这部电影，更确切的说应该是影评吧，一部讲述Facebook创始人Mark Zuckerberg创立公司的电影，当然也有男主角是我很喜欢的Jesse Eisenberg的原因。

这么棒的一个极客电影我居然拖到了现在才看，平时还自诩为一个技术宅呢，真的很让人脸红啊。影片开篇展示的小扎黑掉七个女生宿舍的服务器，然后在四个小时内完成整个facemash的搭建，之后迅速引爆整个哈佛校园网的那一段我真的是张着嘴看完的，不得不佩服天才就是天才。这些事放在我身上我是想都不敢想，何况人当时才大二而我现在已经大四，真的觉得自己就是一个弱逼啊。当然了撸码的速度也被完爆了！

接下来被邀请参与网站建设，看上去很熟悉的一个桥段，因为我也被老师叫去了无数回想要合伙搭建一个平台，什么购物的，社交的……当然我都认为独自一人hold不住拒绝了。而他一句”I’m in“直截了当，回到宿舍几个礼拜就把facebook的第一版敲出来了，我会说这段我是跪着看完的吗？当然不会！

再然后公司扩张招人也是别具特色，通过喝酒敲代码来选出facebook的工程师，也是把facebook的文化展示了出来。就这样一群既狂热又有头脑的疯子聚集在了一起，把facebook发展的越来越壮大。

当然整个影片通过撕逼的方式把这一段历史展示出来也是很有意思的。

影片给我的最大感受就是，小扎的执行力真的是太强的！无关创意，他产生的每一个想法在下一秒就会被他的行动所实现，被甩了，被室友提示女生和畜生比较评分立刻相处女生火辣程度的比拼然后动手四个小时完成，而这个时候其他人都在凤凰社happy。通过校园范围社交网站的点子想出了自己的创意立刻回到宿舍憋了几个星期写出facebook原型并上线，而此时提出想法的三个人还在欣赏合唱和帆船训练。当他开始将facebook的业务扩展到其他学校时兄弟俩还乐呵呵的参加比赛呢。这种执行力真的是恐怖，想到什么就立刻去实现它，个人认为影片中小扎的成功很大的原因是取决于他的恐怖的执行力吧。这也是这部影片带给我的最大收获。

看完影片手痒痒了。

Mac环境下搭建Hexo博客

发表于 2015-12-15   |   分类于 丫一门外汉   |  

Mac环境下搭建Hexo博客

前天刚刚废了好大劲搭建完Wordpress的博客，结果被告知有一键安装版，心情突然很不爽，翻阅其他大神的博客时发现了一个风格很不错的博客Hexo，作为一种静态轻博客易维护易改造，直接放在Github或者Gitcafe上就好，省去了申请域名服务器配置等一大票操作，非常省心。

下面就简要介绍下Mac环境下的安装方法。

安装Node.js

没有Node.js的用户当然要安装下Node.js了，这个直接在官网 http://nodejs.org/ 下载并安装就可以了

安装Git

因为要托管到github上，作为博客的远程仓库，也相当于博客的服务器

没有github帐号的直接申请就可以了，新人比较麻烦的可能在于Ssh Keys的配置，这个百度Ssh git都很容易搜到

安装Hexo

打开命令行控制工具，键入 sudo npm install -g hexo

然后自己建立个站点目录并进入，键入 sudo hexo init

这样本地环境下的hexo博客就算配置好了，cd到你的站点目录，输入hexo generate (hexo g也可以)，然后输入hexo server(hexo s),这事命令行会显示

INFO Hexo is running at http://0.0.0.0:4000/. Press Ctrl+C to stop.

打开浏览器，键入 localhost:4000 就可以看到效果了

部署到Github

首先需要在github上建立与你用户名想对应的仓库，例如我的github用户名为[beastlovezxc],因此我建立的仓库就是[beastlovezxc.github.io]，因此登录到你的github主页上新建个[your_user_name.github.io]的仓库就可以了。

然后回到你的命令行，找到站点目录下的_config.yml文件，键入vi _config.yml
最下面有个deploy，改成以下的格式

deploy:

type: git

repository: https://github.com/your_user_name/your_user_name.github.io.git

branch: master

这里需要注意的是由于文件采用markdown语法，每个冒号后面都要加一个空格，否则会配置失败，本人就是因为在这里疏忽浪费了不少时间，还有一点就是如果hexo版本是3.0以后的type项是git，在3.0前的要改成github，至于如何查看hexo的版本号，cd到你的站点文件夹，然后键入hexo -version即可。

同时注意的一点是你的github已经配置好了你的ssh key。

进行部署

准备工作都已经就绪，接下来在命令行键入hexo deploy(hexo d)即可开始布置你的博客，然后在浏览器中键入http://your_user_name.github.io即可看到你部署到github上的博客了
